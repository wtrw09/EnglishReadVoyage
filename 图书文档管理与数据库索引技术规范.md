# 一、 书籍物理组织规范（物理层）
每本书必须以“独立文件夹”的形式存在，作为一个自包含的资产包。
1. 目录层级：`Books/{Book_Folder_Name}/`
   - (说明：所有书籍文件夹直接存放在 Books 根目录下，不再设物理分级文件夹，分级由数据库逻辑管理。)
2. Book_Folder_Name：建议使用 `ID_书名` 格式，如 `b001_all_about_coyotes`。
3. 必需文件清单：
index.md：书籍的主体内容，使用 --- 进行分页。
meta.json：书籍的元数据真相源（具体见下文）。
cover.jpg：封面图片，建议统一命名，方便程序抓取。
资源管理：
书籍内部引用的图片应统一存放在该书籍目录下的 images/ 文件夹中。
# 二、 meta.json 元数据定义规范
该文件是书籍的身份标识，必须随书移动。
字段 | 类型 | 说明 | 示例
--- | --- | --- | ---
`id` | String | 全局唯一标识符（不可重复） | `"b001"`
`title` | String | 书籍的正式标题 | `"All About Coyotes"`
`author` | String | 作者姓名 | `"Elizabeth Jane Pustilnik"`
`tags` | Array | 静态标签列表（科普、动物等） | `["Science", "Animals"]`
`description` | String | 书籍简短介绍 | `"Learn all about coyotes..."`
`cover_image` | String | 封面文件名 | `"cover.jpg"`
`page_count` | Int | 总页数（可选，扫描时可自动计算） | `12`
# 三、 逻辑层与数据库规范

为了提高检索效率并支持多用户个性化功能，系统引入 SQLite 数据库作为逻辑管理层。

### 1. 数据库建模
数据库负责存储书籍元数据缓存、用户分类信息以及二者的关联关系。详细的表结构定义（含 `users`, `books`, `categories`, `book_category_rel`）请参阅：[数据库设计要求.md](./数据库设计要求.md)。

### 2. 同步与检索原则
- **物理与逻辑解耦**：系统通过 `meta.json` 中的唯一 `id` 建立物理文件与数据库记录的关联。物理路径的变更不应影响用户的收藏与分类状态。
- **多用户隔离**：所有用户生成的分类与关联数据必须锚定 `user_id`，确保不同用户的阅读体验互不干扰。
- **检索加速**：通过数据库索引实现标题、作者、标签等维度的快速过滤与排序。

# 四、 命名规范补充
ID 规范：系统自增。
图片引用：Markdown 中的图片路径应使用相对路径（如 ./images/xxx.jpg），以便解析器正确定位。
