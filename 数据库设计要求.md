# 数据库设计要求

本系统采用 SQLite 数据库进行书籍元数据管理、用户分类及其关联关系的持久化。为了支持多用户环境下的个性化分类与状态，设计遵循以下规范。

## 一、 实体建模规范

### 1. users 表（用户定义）
存储基础用户信息，作为所有个性化数据的归属锚点。
- `id`: INTEGER (PRIMARY KEY AUTOINCREMENT) - 用户唯一标识
- `username`: TEXT (UNIQUE NOT NULL) - 用户名
- `password_hash`: TEXT - 密码哈希
- `created_at`: DATETIME - 注册时间

### 2. books 表（书籍缓存）
存储从物理层 `meta.json` 提取的静态元数据，减少 IO 扫描。
- `id`: TEXT (PRIMARY KEY) - 书籍唯一 ID（对应 meta.json 的 id）
- `title`: TEXT - 书籍标题
- `author`: TEXT - 作者
- `cover_path`: TEXT - 封面图物理路径
- `file_path`: TEXT - 索引文件物理路径
- `page_count`: INTEGER - 书籍总页数
- `sync_hash`: TEXT - 校验值（用于检测 meta.json 变动）
- `last_sync`: DATETIME - 最近一次同步时间

### 3. categories 表（分类定义）
支持系统级预设分类与用户自定义私有分类。
- `id`: INTEGER (PRIMARY KEY AUTOINCREMENT) - 分类唯一标识
- `name`: TEXT - 分类名称（如：科普、已读、我的最爱）
- `type`: TEXT - 类型（`system` 代表系统预设，`user` 代表用户自定义）
- `user_id`: INTEGER (FOREIGN KEY) - 所属用户 ID。对于系统分类，此字段为 NULL。

### 4. book_category_rel 表（关联矩阵）
实现多用户环境下的“一书多类”及个人标记核心逻辑。
- `book_id`: TEXT (FOREIGN KEY) - 关联书籍 ID
- `category_id`: INTEGER (FOREIGN KEY) - 关联分类 ID
- `user_id`: INTEGER (FOREIGN KEY) - 归属用户 ID
- **唯一约束**：`UNIQUE(book_id, category_id, user_id)`

### 5. reading_progress 表（阅读进度）
记录每个用户针对每本书的具体阅读状态。
- `user_id`: INTEGER (FOREIGN KEY) - 用户 ID
- `book_id`: TEXT (FOREIGN KEY) - 书籍 ID
- `current_page`: INTEGER - 当前阅读到的页码（默认 1）
- `last_read_at`: DATETIME - 最后阅读时间（DEFAULT CURRENT_TIMESTAMP）
- `is_completed`: INTEGER - 是否已读完（0 或 1）
- **唯一约束**：`PRIMARY KEY (user_id, book_id)`

## 二、 关键业务逻辑逻辑

### 1. 多用户隔离逻辑
- **分类检索**：查询当前用户的分类列表时，应包含 `user_id` 为当前用户 ID 或 `user_id IS NULL`（系统分类）的所有记录。
- **书籍过滤**：当用户在某个分类下查看书籍时，必须通过 `book_category_rel` 的 `user_id` 字段进行双重过滤，确保 A 用户的分类内容不干扰 B 用户。

### 2. 阅读进度追踪逻辑
- **进度保存**：用户翻页时，实时更新（或根据策略批量更新）`reading_progress` 表中的 `current_page` 和 `last_read_at`。
- **进度读取**：用户再次打开某本书时，后端应优先查询 `reading_progress`，若存在记录则跳转至对应页码。

### 3. 数据同步逻辑
- 系统启动或检测到文件变动时，读取 `meta.json`。
- 若 `sync_hash` 发生变化，仅更新 `books` 表中的元数据。
- 即使物理文件夹重命名（`file_path` 变化），只要 `id` 不变，用户的分类关联（`book_category_rel`）依然有效。

## 三、 性能与扩展建议
- **索引优化**：建议在 `book_category_rel` 的 `user_id` 和 `book_id` 上建立复合索引。
- **扩展性**：后续可在此基础上增加 `user_notes`（个人笔记）表，同样通过 `user_id` 和 `book_id` 进行关联。
